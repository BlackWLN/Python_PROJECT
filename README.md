# Python_PROJECT


"

Цель: Разработка веб-приложения тренажера для подготовки к экзаменам по ФИ и АИСД

Функциональные требования:
1. Регистрация и вход:
   - Регистрация с использованием имени, электронной почты и пароля.
   - Хеширование всех персональных данных, включая пароли, для обеспечения безопасности.
   - Вход в систему с проверкой учетных данных.
   - Регистрация и вход с помощью OAuth.
2. Интерфейс для ввода тестов:
   - Форма для ввода вопросов с вариантами ответов, открытым ответом или расположением ответов в определенном порядке.
3. Интерфейс тестирования:
   - Отображение вопросов по одному в случайном порядке со случайным порядком ответов без хинтов на фронте.


Цель: Расширить функционал веб-приложения, добавив ограничение по времени, ведение статистики, возможность выбора отдельных тем для тестирования (по аналогии с Решу ЕГЭ).

Функциональные требования:
1. Ведение статистики:
   - Возможность просмотра в сравнении результатов текущего и предыдущих тестов, количества вопросов и правильных ответов, общего рейтинга пользователей и своего места в нем.
2. Возможность выбора содержимого тестов:
   - Пользователь может выбирать темы или наборы тем, по которым он хотел бы проходить тестирование.
3. Подсказки для пользователя:
   - Предлагать пользователю темы или наборы тем вопросов на основании предыдущей статистики ответов пользователя, выбирая темы, в которых пользователь не проявил себя.
4. Ограничение по времени:
   - Реализовать ограничение по времени на весь тест или на один вопрос.

Цель: Добавить возможность для пользователей оценивать вопросы и тесты, предлагать собственные вопросы, элементы геймификации и интеграции с другими медиа.

Функциональные требования:
1. Интеграция с Телеграм:
   - Реализация Телеграм-бота для тестирования непосредственно в приложении.
   - Интеграция статистики и элементов геймификации в бот, а также возможности выбора тем вопросов.
2. Оценка тестов пользователем:
   - Реализовать для пользователя возможность предлагать изменения правильных вариантов ответа или изменение текста вопроса, давать фидбек по вопросу и/или тесту в целом.
3. Добавление вопросов пользователя:
   - Реализовать возможность для пользователя предлагать свои собственные вопросы с вариантами ответов
4. Элементы геймификации:
   - Добавить систему ачивок для тестируемых с уровнями упорства/экспертности."
